# T-RO 방향성 및 작성 지침 요약

## 1. 저널의 성격과 방향성

**IEEE Transactions on Robotics (T-RO)**는 로봇 분야의 대표 저널 중 하나로, 로봇 시스템, 알고리즘, 이론, 실제 적용까지 폭넓게 다룬다.  
이 저널에서는 단순한 알고리즘 소개보다 **로봇 문제 자체의 중요성**, **로봇 시스템 관점에서의 설계**, **실험을 통한 설득력 있는 검증**을 매우 중요하게 본다.

### 핵심 성격
- 로봇 이론과 실제 시스템을 함께 다룸
- 로봇 시스템 관점의 연구를 선호
- robotics relevance가 매우 중요
- 실험 기반 검증 요구가 강함

### T-RO 논문에 일반적으로 요구되는 요소
- 명확한 robotics problem 정의
- 새로운 방법론
- 충분한 실험
- 실제 로봇 또는 현실성을 충분히 의식한 시뮬레이션 검증

## 2. 논문 형식

T-RO 논문은 IEEE Transactions 계열 저널의 공통 형식을 따른다.

### 기본 레이아웃
- **2단 형식**
- **IEEEtran LaTeX class 사용**
- **10pt 글꼴**
- **8.5×11 inch 페이지**

### 첫 페이지 구성
첫 페이지에는 일반적으로 다음 내용이 바로 포함된다.

- Title
- Author names / affiliations
- Abstract
- Keywords
- 본문 시작

별도의 표지 페이지는 사용하지 않는다.

## 3. Abstract 작성 규칙

Abstract는 논문 전체를 매우 압축해서 보여주는 부분이다.

### 기본 규칙
- **100–250 words**
- **하나의 문단**
- **참고문헌 인용 금지**
- **수식 포함 금지**
- 논문의 전체 핵심을 요약해야 함

### 보통 포함해야 하는 내용
- Problem
- Proposed approach
- Key result
- Significance

## 4. 논문 길이

Transactions 저널 기준으로 보면 일반적으로 다음 범위를 생각할 수 있다.

- 보통 약 **10~12 pages**
- 2단 형식 기준
- 페이지가 늘어나면 추가 비용이 붙을 수 있음

로봇 저널 문맥에서는 보통 **10~20 pages** 정도가 일반적인 범위로 볼 수 있다.  
따라서 **10~20페이지 목표**는 적절한 설정이다.

## 5. Figure와 그래픽 규칙

IEEE Transactions 논문에서는 그림의 완성도가 매우 중요하다.

### 기본 규칙
- 모든 figure에는 번호가 있어야 함
- 본문에서 반드시 참조되어야 함
- 축, 범례, 텍스트가 너무 작아지지 않도록 설계해야 함

### 중요한 작성 관례
T-RO 논문에서는 보통 **Figure 1이 논문의 전체 아이디어를 설명하는 그림** 역할을 한다.

자주 쓰이는 구조는 다음과 같다.

- problem
- idea
- pipeline
- RL training 또는 전체 시스템 흐름

즉 Figure 1은 단순 예시 그림이 아니라, 논문 전체를 한 장으로 요약하는 핵심 그림이 되는 경우가 많다.

## 6. Reference 스타일

참고문헌은 **IEEE 스타일**을 사용한다.

### 본문 인용 방식
- `[1]`
- `[2]`
- `[3]`

### 참고문헌 표기 예시
- `A. Author, “Paper title,” Journal, vol., pp., year.`

즉 번호 기반 인용 체계를 사용하며, 본문과 참고문헌 목록 모두 IEEE 형식에 맞춰야 한다.

## 7. 전체적으로 보면 T-RO 논문에서 중요한 것

형식 규칙도 중요하지만, 실제로 더 중요한 것은 아래 네 가지다.

- **로봇 문제를 분명히 제시하는 것**
- **방법론의 새로움이 분명한 것**
- **실험 검증이 충분한 것**
- **실제 로봇 또는 현실성 있는 시뮬레이션 맥락을 갖추는 것**

즉 T-RO 논문은 단순히 형식을 맞추는 것을 넘어서,  
**로봇 분야에서 왜 중요한 문제인지**,  
**무엇이 새롭고**,  
**실험으로 얼마나 설득력 있게 보였는지**가 핵심이다.
